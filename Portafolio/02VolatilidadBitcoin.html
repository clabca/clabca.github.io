# Importar bibliotecas
import yfinance as yf
import pandas as pd
from arch import arch_model
import numpy as np

# Obtener datos histricos de Bitcoin desde Yahoo Finance
btc_data = yf.download('BTC-USD', start='2020-01-01')

# Calcular los retornos diarios
btc_returns = btc_data['Close'].pct_change().dropna()

# Especificar el modelo GARCH(1,1)
garch_model = arch_model(btc_returns, vol='Garch', p=1, q=1)

# Ajustar el modelo
garch_fit = garch_model.fit()

# Realizar pronstico de la volatilidad futura
forecast_horizon = 10
vol_forecast = garch_fit.forecast(start=0, horizon=forecast_horizon)

# Definir el nivel de confianza
conf_level = 0.05

# Calcular el z-score para el nivel de confianza
z_score = np.percentile(btc_returns, 100*(1-conf_level))

# Calcular el VaR
VaR = z_score * np.sqrt(forecast_horizon) * vol_forecast.variance.values[-1]**0.5

print(f'El VaR a un nivel de confianza del {conf_level*100}% es: {VaR[0]:.2f}')

import yfinance as yf
import pandas as pd
from arch import arch_model
import numpy as np
import seaborn as sns
import matplotlib.pyplot as plt

# Obtener datos histricos de Bitcoin desde Yahoo Finance
btc_data = yf.download('BTC-USD', start='2020-01-01')
btc_returns = btc_data['Close'].pct_change().dropna()

# Calcular volatilidad histrica
vol_historica = btc_returns.rolling(window=20).std() * np.sqrt(252)

# Especificar modelo GARCH(1,1)
garch_model = arch_model(btc_returns, vol='Garch', p=1, q=1)
garch_fit = garch_model.fit()

# Realizar pronstico de la volatilidad futura
forecast_horizon = 10
vol_forecast = garch_fit.forecast(start=0, horizon=forecast_horizon)

# Extraer la volatilidad pronosticada
vol_pronosticada = np.sqrt(vol_forecast.variance.values[-1] * 252)

# Convertir los valores de volatilidad a numricos
vol_historica = vol_historica.iloc[-1]
vol_pronosticada = vol_pronosticada[0]

# Crear DataFrame para los datos
data = {
    'Tipo de Volatilidad': ['Volatilidad Histrica', 'Volatilidad Pronosticada'],
    'Volatilidad Anualizada': [vol_historica, vol_pronosticada]
}

df = pd.DataFrame(data)

# Crear el grfico con Seaborn
plt.figure(figsize=(10, 6))
sns.barplot(x='Tipo de Volatilidad', y='Volatilidad Anualizada', data=df, palette='pastel')
plt.title('Volatilidad Histrica vs. Volatilidad Pronosticada del Bitcoin')
plt.show()

Conclusiones¶

Volatilidad del Bitcoin:¶

La volatilidad del Bitcoin ha sido una caracterstica prominente durante el perodo analizado. Esto implica que el precio del Bitcoin puede experimentar cambios significativos en un corto perodo de tiempo, lo que puede representar tanto oportunidades como riesgos para los inversores.

Modelo GARCH(1,1):¶

El modelo GARCH(1,1) se utiliz para modelar la volatilidad del Bitcoin. Este modelo es una herramienta til para entender y prever la volatilidad de los activos financieros.

Volatilidad Histórica vs. Volatilidad Pronosticada:¶

Comparando la volatilidad histrica con la volatilidad pronosticada, se observa que la volatilidad pronosticada puede proporcionar una estimacin valiosa de la volatilidad futura. Esto es esencial para la gestin de riesgos y la toma de decisiones de inversin.

Valor en Riesgo (VaR):¶

Se calcul el Valor en Riesgo al 5% de confianza. Esto proporciona una estimacin de la prdida mxima que se puede esperar en un perodo determinado con un nivel de confianza del 95%.

Estrategias de Gestión de Riesgos:¶

Basndose en el VaR calculado, los inversores pueden desarrollar estrategias de gestin de riesgos para proteger sus inversiones ante movimientos adversos del mercado.

Importancia de la Diversificación:¶

Dada la alta volatilidad del Bitcoin, es importante que los inversores diversifiquen sus carteras con otros activos para reducir el riesgo.

Evolución Temporal:¶

El grfico de volatilidad histrica a lo largo del tiempo muestra cmo la volatilidad del Bitcoin ha fluctuado en el perodo analizado. Esto puede proporcionar informacin valiosa sobre patrones y tendencias a lo largo del tiempo.

Limitaciones del Modelo:¶

Es importante recordar que el modelo GARCH(1,1) es una simplificacin y no tiene en cuenta todos los factores que pueden influir en la volatilidad del Bitcoin.

Continua Supervisión:¶

Dado que los mercados financieros son dinmicos y estn sujetos a cambios, es crucial mantener una supervisin continua y realizar ajustes en las estrategias segn sea necesario.
